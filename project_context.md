## Git Workflow Requirements

1. Every completed module must result in a git commit.
2. Suggest a professional commit message after generating code.
3. Follow Conventional Commits:
   - `feat:`
   - `fix:`
   - `docs:`
   - `refactor:`
   - `chore:`
4. Do not bundle unrelated changes into one commit.
5. After each feature completion:
   - `git add .`
   - `git commit -m "<message>"`
   - `git push origin main`
6. Maintain a clean commit history suitable for assessment review.
